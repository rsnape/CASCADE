public final class Consts

AGGREGATOR_PROFILE_BUILDING_PERIODE

public final static int AGGREGATOR_PROFILE_BUILDING_PERIODE
This is the aggregator profile building period time in terms of day, assuming it is set manually During this period, aggregator do not send any signal and simply get the the normal usage of it customers baseline demand and at the end of this period, it will build an average usage profile, which will used during training period. This period should at least take sometimes about 3 to 7 days.

AGGREGATOR_TRAINING_PERIODE

public final static int AGGREGATOR_TRAINING_PERIODE
This is the aggregator training period time in terms of day (assuming it is set manually) During this time, aggregator send S=1 signal to test customers behaviors and hence estimate k and e. At minimum, it needs to be 48 days in order to complete sending S=1 signals, one for each 48 timeslote of the day. This time is above the time that aggregator needs to gather and build an average baseline profile of its customers (prosumers), usually between 4-7 days
